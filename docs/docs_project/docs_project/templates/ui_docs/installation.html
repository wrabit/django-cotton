Installation Set up Cotton UI in your Django project with Tailwind CSS and Alpine.js. Requirements Install Packages Django Settings Base Template Custom build (optional) Theming & verify

Cotton UI needs:

  • Python 3.8+
  • Django 4.2+
  • No build step to use the components, the kit ships a precompiled stylesheet you <link>, and you theme it by overriding CSS variables.
  • Tailwind CSS v4 is optional, only if your own pages use Tailwind utility classes (the kit never needs it).
  • Alpine.js v3 for interactive components (loaded from a CDN below).

Install via pip:

{% cotton:verbatim %}{% verbatim %} pip install django-cotton-ui {% endverbatim %}{% endcotton:verbatim %}

This will also install django-cotton as a dependency.

Add both django_cotton and django_cotton_ui to your INSTALLED_APPS:

{% cotton:verbatim %}{% verbatim %} INSTALLED_APPS = [ # ... "django_cotton", "django_cotton_ui", # ... ] {% endverbatim %}{% endcotton:verbatim %}

Order matters: django_cotton must come before django_cotton_ui.

Link the kit's precompiled stylesheet and the Alpine bundle in your base template, then brand it by overriding the design tokens right in the <head>:

{% cotton:verbatim %}{% verbatim %} {% load static %} {{ slot }} {% endverbatim %}{% endcotton:verbatim %}

Both files ship inside the installed package. With django.contrib.staticfiles enabled, {% verbatim %}{% static %}{% endverbatim %} resolves them in development; run collectstatic for production. Nothing to copy or build.

Required Alpine.js Plugins

Components need two Alpine.js plugins, loaded before Alpine core:

  • @alpinejs/collapse - Accordions, the collapse component and the mobile navbar menu
  • @alpinejs/focus - Focus trapping in dialogs and drawers, and keyboard focus in dropdowns and menus

The precompiled stylesheet covers every component class, and theming is just token overrides (the <style> block above), so a build is optional. You only need one if your own pages use Tailwind utility classes, as most projects do. If you run one, scan only your own templates and link the output after the kit's stylesheet. The @custom-variant dark line is required, it keeps your build's class-based dark mode in step with the kit:

Without this your build's dark: utilities default to a media query, which misfires and overrides the kit's own dark: rules. */ @custom-variant dark (&:where(.dark, .dark *)); /* Scan only YOUR templates.
