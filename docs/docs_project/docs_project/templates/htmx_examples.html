Working with HTMX

Cotton components work seamlessly with HTMX, serving a dual purpose: they can be used as reusable components in your templates AND as HTMX partial responses from your views.

This approach follows the "pure Cotton" pattern - no need for django-template-partials or other libraries. Simply render your Cotton components directly from your views.

The Pure Cotton Pattern:

  • Create a Cotton component in cotton/component-name.html
  • Use it in templates with Cotton syntax
  • Return it from views: render(request, 'cotton/component-name.html', context)
  • HTMX swaps it into the DOM seamlessly

    Key Takeaways

  • Cotton components work seamlessly as HTMX partial responses
  • No need for django-template-partials or special configuration
  • Use {% verbatim %}{{ attrs }}{% endverbatim %} to preserve HTMX attributes
  • Components maintain their reusability while serving dynamic content
  • Combine with Alpine.js for enhanced interactivity
