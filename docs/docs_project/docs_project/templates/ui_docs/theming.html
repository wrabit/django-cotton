Cotton UI uses a Flux-inspired theming approach with two primary colors:

  • Base Color (zinc) - Used for backgrounds, borders, and text throughout the UI
  • Accent Color (teal) - Used for primary buttons, active states, and focus rings

Both colors are fully customizable using CSS variables, with no Tailwind rebuild required.

Changing the Accent Color

The accent color is used for primary interactive elements like buttons, tabs, and focus states. Cotton UI provides a 3-value accent system:

  • accent - Base accent color
  • accent-content - Stronger variant for hover states
  • accent-foreground - Text color on accent backgrounds
{% cotton:verbatim %}{% verbatim %} /* In your CSS file */ @layer base { :root { /* Change accent from teal to blue */ --color-accent-base: var(--color-blue-500); --color-accent-content-base: var(--color-blue-600); --color-accent-foreground-base: var(--color-white); } .dark { --color-accent-base: var(--color-blue-400); --color-accent-content-base: var(--color-blue-300); --color-accent-foreground-base: var(--color-blue-950); } } {% endverbatim %}{% endcotton:verbatim %}

Accent Props

Cotton UI components support an :accent prop to opt out of accent colors for a monochrome style. By default, components use accent colors (:accent="True"), but you can set :accent="False" to use zinc-based colors instead.

When :accent="False", selected states use zinc-900 (light mode) or zinc-100 (dark mode) instead of the accent color. This creates a subtle, monochrome aesthetic that works across radios, checkboxes, and other interactive components.

Changing the Base Color

Cotton UI uses zinc as the default base color for grays. You can swap it for another gray family like slate, gray, or neutral by remapping all zinc shades:

{% cotton:verbatim %}{% verbatim %} /* Remap zinc to slate */ @theme { --color-zinc-50: var(--color-slate-50); --color-zinc-100: var(--color-slate-100); --color-zinc-200: var(--color-slate-200); --color-zinc-300: var(--color-slate-300); --color-zinc-400: var(--color-slate-400); --color-zinc-500: var(--color-slate-500); --color-zinc-600: var(--color-slate-600); --color-zinc-700: var(--color-slate-700); --color-zinc-800: var(--color-slate-800); --color-zinc-900: var(--color-slate-900); --color-zinc-950: var(--color-slate-950); } {% endverbatim %}{% endcotton:verbatim %}

How Components Use Colors

Components use semantic color utilities that automatically reference your theme:

  • bg-accent - Accent background (e.g., primary buttons)
  • text-accent-foreground - Text on accent backgrounds
  • hover:bg-accent-content - Accent hover state
  • ring-accent - Focus rings use accent color
  • bg-zinc-100 - Base gray backgrounds (full 50-950 range available)
  • border-zinc-300 - Base gray borders
  • text-zinc-700 - Base gray text

Based on Flux UI's theming approach — simple, flexible, and no rebuild required.
