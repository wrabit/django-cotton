Why cotton?

{% heroicon_outline 'fire' class="shrink-0 text-teal-600 size-16 mr-5 mt-1" stroke_width="1.8" %} Create reusable UI components from a single HTML file, no Python required. Composing from small, modular pieces streamlines your workflow and boosts productivity. Build components → Components own their markup, so you can style them however you like: plain CSS, your design system or utility frameworks like Tailwind.
{%
 Cotton enhances Django templates without replacing them, allowing progressive enhancement while maintaining full use of existing template features. {% heroicon_outline 'code-bracket' class="shrink-0 text-teal-600 size-16 mr-5 mt-1" stroke_width="1.8" %} Cotton's optional HTML tag-like syntax allows code editors to recognize its components as HTML elements, enabling features like syntax highlighting and automatic tag completion. Tag syntax → {% heroicon_outline 'presentation-chart-line' class="shrink-0 text-teal-600 size-16 mr-5 mt-1" stroke_width="1.8" %} Cotton compiles to native Django template tags and caches the result, so it adds virtually no runtime overhead. {% heroicon_outline 'bolt' class="shrink-0 text-teal-600 size-16 mr-5 mt-1" stroke_width="1.8" %} Components are just templates, so they drop straight into HTMX swaps and pair naturally with Alpine.js for rich, server-driven interactivity. Why not just Django templates?

You still can, and Cotton stays fully interoperable with them. But the native tools for reuse get verbose fast. Here is the same idea, the native way and with Cotton.

Native DTL: Verbose & Limited

{% cotton:verbatim %}{% verbatim %} {# ❌ include: HTML is escaped #} {% include "card.html" with content="Bold..." %} {# ❌ extends: only one per template #} {% extends "card.html" %} {% block content %}Card 1{% endblock %} {% extends "card.html" %} {% block content %}Card 2{% endblock %} {# ❌ templatetags: needs a .py file #} {% load card %} {% card %} {% endverbatim %}{% endcotton:verbatim %}

Cotton: Composable & Re-usable

{% cotton:verbatim %}{% verbatim %} {# ✅ pass rich HTML content #} Bold, links, anything {# ✅ reuse unlimited times #} ... ... {# ✅ just an alert.html file, no Python #} No Python needed {% endverbatim %}{% endcotton:verbatim %}
